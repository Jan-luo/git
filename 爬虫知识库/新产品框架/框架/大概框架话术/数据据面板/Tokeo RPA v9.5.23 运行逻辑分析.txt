一、项目摘要

基于 WebView2 + Win32 窗口操控的社交媒体自动化 RPA 工具，针对抖音/小红书评论
区抓取潜在客户 → 自动私信营销 → AI 对话引导留资 → 微信承接转化。私有化部署、
脚本驱动、参数灵活。

三、自动化流水线（5 步）

阶段1 - 监听（WebView2 隐身浏览器）
  · WebView2 以无头/最小化模式加载抖音/小红书网页版
  · 通过 AddScriptToExecuteOnDocumentCreated 在页面 DOM 构建前注入 JS
  · 劫持滚动事件自动翻评论、劫持 DOM 插入捕获新评论
  · 劫持 XMLHttpRequest.send / fetch 抓取 API 返回的用户数据（昵称/地区/历史评论）

阶段2 - 匹配（本地规则引擎）
  · 评论分词 → 匹配 关键词.txt（想来/多少钱/攻略/报名 等 ~200条）
  · 命中 → 检查 二次筛选.txt 排除词（已经/可惜/留言/无偿 等）
  · 排除词命中 → 跳过（skipData++）
  · 排除词未命中 → 标记为潜在客户（successData++）
  · 从 API 返回数据提取省份字段（用于数据看板地域统计）

阶段3 - 触达（双通道并行）
  通道A（私信）：WebView2 DOM 操作点击私信按钮
    → 从 私信话术.txt 随机选1条 + 从 随机表情.txt 随机选1个 → 拼接发送
    → 发送间隔：15-35秒（config.ini 分段控制）
  通道B（评论提醒）：WebView2 操作评论区 @用户名
    → 从 提醒话术.txt 随机选1条发送

阶段4 - 微信承接（Win32 API 窗口操控）
  · FindWindow(class="Qt51514QWindowIcon", title="文件传输助手") 定位微信
  · SendInput / keybd_event 模拟键入：搜索框 → 微信号 → 添加通讯录 → 验证信息
  · 操作间隔由 UK=3 控制（秒级），模拟人类速度

阶段5 - AI 对话（角色模型驱动）
  · 加载 角色模型.txt 决策树
  · 内部静默识别联系方式（11位数字→手机号，字母+数字→微信号）
  · 输出绝不暴露"已识别" → 按场景分支引导留资


四、AI 角色模型决策树

场景A：客户留了联系方式 → "旅行管家稍后联系您"（不暴露已识别）
场景B：客户不想留联系     → 三轮渐进话术：免费规划 → 案例参考 → 一对一服务
场景C：客户要求直接发行程  → 拒绝，引导先留微信
场景D：客户要求加客服微信  → 拒绝（"公司规定不能主动加"），引导对方留号
场景E：客户多次拒绝       → 停止索要

禁止行为：
  · 不得主动发送自己的微信或电话
  · 不得在没有获取联系方式前提供详细行程
  · 不得与客户争论或强行推销


五、核心技术细节

多线程并发：thread=15，15个并发线程处理不同账号/平台
账号隔离  ：每个账号独立 WebView2 UserDataFolder（Cookie/Token 不共享）
频率控制  ：A=36000(10h主循环), 1=15s, 2=35s, 3=10s, 4=25s(分段发送间隔)
           5=360s, 6=6s, 7=6s, 8=18s(账号轮换间隔), UK=3s(微信操作间隔)
地域定向  ：config.ini 中多处 湖南| 标记，按地域筛选目标客户
异常恢复  ：程序崩溃自动重启，网络断开自动重连


六、数据看板

Analysis.html 数据管道：
  commentData() → successData() → failData() → skipData()
  → totalProcessed / successRate / failRate / skipRate
  → province（名称正则提取省份） → yesterdayTotal（逐日对比）
  → ECharts 渲染：省份热力图 + 成功率饼图 + 逐日趋势线 + 实时日志

八、旧版模块对照（模板旧版目录）

旧版 Tokeo RPA 界面分为以下功能模块，与新版的 Script 配置一一对应：

  AI调试    → 角色模型.txt（AI 对话决策树配置调试界面）
  作品监听   → WebView2 监听引擎界面（抖音/小红书评论实时捕获）
  博主采集   → WebView2 博主主页爬取界面
  客户数据   → Analysis.html 数据看板界面
  搜索采集   → 关键词.txt + 二次筛选.txt 配置界面
  数据监视   → 实时操作日志监控界面
  综合拓客   → 主流程控制面板（启停/模式切换/账号管理）
  群组采集   → 群组/话题批量抓取界面
  记录查询   → 历史操作记录搜索界面
  词库配置   → 话术模板 + 关键词库编辑管理界面（截图最多，功能最复杂）

新版 v9.5.23 将这些界面整合为统一主程序 + Script 配置文件 + src 数据看板。
